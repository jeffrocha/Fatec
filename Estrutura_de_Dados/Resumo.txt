Arrays (Vetores): Uma coleção de elementos do mesmo tipo, acessados por um índice. Acesso é rápido, mas a inserção e remoção podem ser custosas, dependendo da posição.

Listas Ligadas: Uma sequência de elementos, onde cada elemento (nó) contém o valor e uma referência ao próximo nó. Inserções e remoções podem ser eficientes, mas o acesso a elementos requer percorrer a lista.

Pilhas: Uma estrutura de dados LIFO (último a entrar, primeiro a sair), onde os elementos são adicionados e removidos apenas no topo. É comumente usado para rastrear estados ou para reverter ações.

Filas: Uma estrutura de dados FIFO (primeiro a entrar, primeiro a sair), onde os elementos são adicionados no final e removidos no início. É usado em situações que exigem processamento em ordem de chegada.

Árvores: Uma estrutura hierárquica de elementos chamados nós, onde um nó é considerado a raiz e os outros são seus filhos. Existem várias variações, como árvores binárias, árvores de busca e árvores balanceadas.

Grafos: Uma coleção de nós conectados por arestas, onde os nós podem representar entidades e as arestas suas relações. Grafos podem ser direcionados ou não direcionados.

Tabelas Hash: Uma estrutura que mapeia chaves para valores, permitindo acesso rápido aos valores com base nas chaves. É eficiente para busca, inserção e remoção, mas pode ter colisões que precisam ser resolvidas.

Heap: Uma estrutura de dados de prioridade que mantém os elementos de forma que o elemento de maior (ou menor) prioridade possa ser acessado rapidamente.

Matrizes: Uma estrutura bidimensional que organiza os elementos em linhas e colunas. É usado para representar informações tabulares.
